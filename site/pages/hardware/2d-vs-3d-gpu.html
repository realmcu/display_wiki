2D / 2.5D / 3D GPU 技术对比报告

摘要

本报告对比分析嵌入式显示领域的三类图形处理器（2D GPU、2.5D GPU、3D GPU），从技术原理、功能特性、性能指标、应用场景等维度进行系统性对比，为显示方案选型提供技术参考。

1. 基础概念扫盲

1.1 什么是 GPU？

GPU（Graphics Processing Unit，图形处理器）是专门用于图形渲染的处理器。与 CPU 不同，GPU 擅长并行处理大量简单计算，特别适合图形绑制。

1.2 三类 GPU 的定位

类型

定位

形象比喻

2D GPU

平面图形加速

在纸上画画

2.5D GPU

平面图形 + 变形效果

折纸、翻书页

3D GPU

真正的三维渲染

搭建乐高模型

1.3 核心区别一句话总结

  • 2D GPU：只能画平面图形

  • 2.5D GPU：能让平面图形变形，看起来有立体感，但不是真 3D

  • 3D GPU：能渲染真正的三维物体，有光照、阴影、材质

3.3 贴图效果示例：渲染砖墙

2.5D GPU 方案：

输入：1 张砖墙照片（含固定光照）
处理：贴到四边形 → 可旋转/倾斜
输出：平面砖墙，光照固定不变

3D GPU 方案：

输入：3 张图片
  ├─ 颜色图（砖墙纯色）
  ├─ 法线图（凹凸信息）
  └─ 粗糙度图（反光信息）
处理：GPU 实时计算光照
输出：逼真砖墙，光源移动时阴影跟着变化

4. 架构原理对比

4.1 渲染管线

2D GPU：

CPU 命令 → 2D 引擎 → 帧缓冲 → 显示
              │
           DMA 控制器

2.5D GPU：

CPU 命令 → 变换引擎 → 纹理单元 → 混合单元 → 帧缓冲 → 显示
              │           │
           顶点缓存    纹理缓存

3D GPU：

CPU 命令 → 顶点着色器 → 图元装配 → 光栅化 → 片段着色器 → 混合 → 帧缓冲 → 显示
              │            │          │           │
           顶点缓存     几何缓存   纹理缓存    深度缓存

8.2 选型检查清单

选择 2D GPU 的条件：

  • ✅ 纯 2D 界面（按钮、文字、图标）

  • ✅ 功耗要求 < 100mW

  • ✅ 成本敏感

  • ✅ 开发周期短

选择 2.5D GPU 的条件：

  • ✅ 需要旋转、透视、倾斜效果

  • ✅ 需要矢量图形（SVG、OpenVG）

  • ✅ 功耗要求 < 500mW

  • ✅ 不需要真实光照

选择 3D GPU 的条件：

  • ✅ 需要 3D 模型渲染

  • ✅ 需要动态光照和阴影

  • ✅ 需要可编程着色器

  • ✅ 可接受较高功耗（> 500mW）

9. 总结

9.1 核心结论

  1. 2D GPU：适合简单 UI，功耗最低，成本最低

  2. 2.5D GPU：适合需要动画效果但不需要真 3D 的场景，性价比最高

  3. 3D GPU：适合需要真实 3D 渲染的场景，功能最强但成本最高

9.2 关键记忆点

问题

答案

2.5D 和 3D 最大区别？

2.5D 没有光照计算，3D 有

2.5D 能做什么？

旋转、透视、矢量图形

2.5D 不能做什么？

真实光照、阴影、复杂 3D 模型

3D GPU 需要贴图吗？

需要，而且可以用多张贴图叠加

手表该用哪种？

2.5D GPU（够用且省电）

导航该用哪种？

3D GPU（需要 3D 地图）
